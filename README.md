# Local FAERS Pharmacovigilance Pipeline

## What's in this folder

| File | Purpose |
|------|---------|
| `benchmarking_txgnn_on_ntds_4th_biotech_research.py` | Original Colab notebook (do not run directly) |
| `patch_for_local.py` | One-time converter ‚Äî creates `run_local.py` |
| `requirements.txt` | All Python dependencies, M1-safe |
| `run_local.py` | **‚Üê generated by patcher; this is what you run** |

---

## Step 1 ‚Äî Python environment (do this once)

Open VS Code's integrated terminal (`Ctrl+\``) and run:

```bash
# Create a dedicated virtual environment
python3 -m venv .venv

# Activate it (run this every new terminal session)
source .venv/bin/activate

# Install all dependencies
pip install --upgrade pip
pip install -r requirements.txt
```

> **M1 note:** PyTorch ships native arm64 wheels since v2.0 ‚Äî no special flags needed.  
> If `pip install torch` stalls, try: `pip install torch --index-url https://download.pytorch.org/whl/cpu`

---

## Step 2 ‚Äî Generate the clean local script (do this once)

```bash
python patch_for_local.py
```

This reads the original Colab `.py`, strips all `google.colab`, `IPython.display`,
`!pip install`, and `/content/drive/` references, and writes `run_local.py`.  
All `plt.show()` calls are converted to `plt.close()` ‚Äî figures are saved to
`./figures/` instead of popping up windows.

---

## Step 3 ‚Äî Run the pipeline

```bash
python run_local.py
```

### What happens on first run
1. **API fetch** (~5‚Äì10 min): queries the openFDA REST API for all 6 disease cohorts.
   Results are cached in `./data/faers_parsed.pkl` and `./data/faers_exploded.pkl`.
2. **Analysis**: PRR, ROR, BCPNN IC scores, NLP clustering, all tiers.
3. **Figures**: saved as PNG to `./figures/`.
4. **Tables**: saved as CSV to `./tables/`.

### Subsequent runs (cached)
After the first run the pickles exist locally, so the script loads from disk and
skips the API entirely ‚Äî runs in seconds.

---

## Step 4 ‚Äî Run interactively in VS Code (optional, recommended)

VS Code can run this file **cell by cell** like a notebook using the
**Jupyter interactive window**:

1. Install the **Python** and **Jupyter** extensions in VS Code.
2. Open `run_local.py`.
3. Press `Shift+Enter` on any line ‚Äî VS Code detects the `"""..."""` cell markers
   and executes the current cell.

This gives you inline outputs (DataFrames, print statements) without needing
to run the entire 4 000-line file at once.

## Troubleshooting

### `ModuleNotFoundError: No module named 'google.colab'`
You're running the *original* file. Run `patch_for_local.py` first to generate
`run_local.py`.

### `matplotlib` opens no windows / figures look blank
This is expected ‚Äî the script uses the `Agg` (non-GUI) backend so it never
blocks. All figures are saved as high-res PNGs in `./figures/`.  
To view them: `open ./figures/` in Finder, or use VS Code's file explorer.

### openFDA 429 / rate limit errors
The script sleeps 0.3 s between requests. If you hit 429s add a longer sleep:
in `run_local.py` find `time.sleep(0.3)` and change it to `time.sleep(1.0)`.

### Age values like `221 years`
Known FAERS data quality issue ‚Äî some ages are reported in months or days,
not years. Flagged as a limitation in the manuscript methods section.

### `torch` slow on CPU
The NLI models (DeBERTa, PubMedBERT) run on CPU by default. On M1 you can
enable the **MPS** backend for ~3‚Äì5√ó speedup. In `run_local.py`, after the
`import torch` line, add:
```python
device = "mps" if torch.backends.mps.is_available() else "cpu"
```
Then pass `device=device` when loading the pipeline:
```python
pipe = pipeline(..., device=device)
```
